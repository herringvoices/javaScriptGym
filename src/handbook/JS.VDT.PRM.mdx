<Callout type="note" title="Mental model">Variables are <strong>boxes you make so your program can remember something</strong>. You build a box, give it a label, and put something inside. Later you use the label to look up what’s there.</Callout>

Welcome! Today you’ll meet the first big idea in JavaScript: **variables**. A variable is a *box* you make in code so your program can **remember** something—like a player’s score, a visitor’s name, or how many items are in a cart.

Keep this pocket model close: **Variables are boxes.** You build a box, give it a label, and put something inside. Later you use the label to look up what’s in the box. We’ll gently turn that idea into real JavaScript.

### In this chapter, you’ll learn (VDT.PRM)

* **JS.VDT.PRM.1 — Build and label boxes (variables):** How to *declare* variables with `let` and `const`, when to choose a box you can change (`let`) vs. a box you promise not to re‑label (`const`).
* **JS.VDT.PRM.2 — What goes in the box (primitives):** How to store and use **strings** (text), **numbers** (math), **booleans** (true/false), and the two kinds of “empty”: `null` (empty on purpose) and `undefined` (not assigned yet).
* **JS.VDT.PRM.3 — Work with values using operators:** How to do **arithmetic** with numbers, **concatenation** and **interpolation** with strings, and a gentle peek at **comparisons** (`===`, `>`, etc.) and **logical operators** (`&&`, `||`, `!`) to produce `true`/`false`—without jumping into `if/else` yet.
* **JS.VDT.PRM.4 — Pick the right type:** How to decide whether a value should be a **string**, **number**, or **boolean**, and when to use `null` for “nothing yet.”

## 1) Building a box

To build a box in JavaScript, you use the word `let`. `let` is a **keyword** (a reserved word with special meaning) that tells JavaScript: “Create a new box for me.”

```js
let score;
```

You just made a box named `score`. It’s empty right now. To put something inside, use the `=` sign (that’s **assignment**):

```js
score = 0;
```

If you already know what should go inside when you build the box, you can do both in one step:

```js
let score = 0;
```

From now on, whenever you write `score` in your code, JavaScript will hand you whatever is inside that box at that moment.

### Technical Vocabulary

| Term | Beginner-friendly definition | Quick example |
| :---- | :---- | :---- |
| **Variable** | A named **box in memory** that holds a value you can use later. | `let score = 0;` |
| **Declare** | **Build the box.** With `let` or `const`. | `let score;` |
| **Assign** | **Put a value in the box** or replace what’s inside using `=`. | `score = 0;` |

## 2) Labeling your box (naming)

Good labels make code easier to read.

* Must start with a **letter**; can include letters/numbers/`$`/`_` after
* JavaScript cares about **capitalization**: `username` vs `userName`
* Use **camelCase** for multi‑word names: `totalPrice`, `firstName`
* Be specific: `cartTotal` is clearer than `x`

```js
let userName = "Momo";
let cartTotal = 12.50;
```

## 3) What can go in the box? (Primitives)

JavaScript has five simple data types called **primitives**. A **data type** tells you what kind of value is in the box *and* what operations make sense.

### Strings: Your Word Boxes

Strings hold **text**. Use quotes: double, single, or backticks.

```js
let greeting = "Hello";
```

Glue strings together with `+` (**concatenation**):

```js
let first = "Nick";
let message = "Hello, " + first + "!"; // "Hello, Nick!"
```

Important: `"2"` is a **string** while `2` is a **number**.

```js
"2" + 2;                // "22"
Number("2") + 2;        // 4
parseInt("2", 10) + 2;  // 4
```

➡️ **Interpolation** (string templates) uses backticks and `${ }` to drop variables/expressions straight into a string:

```js
let message2 = `Hello, ${first}!`;
let price = 2; let quantity = 3;
let totalLine = `You bought ${quantity} items for ${price * quantity} dollars.`;
```

*When to use strings:* names, labels, messages, IDs not used for math.

### Numbers: Your Math Boxes

One number type for whole + decimal values.

```js
let lives = 3;
let price2 = 2.99;
let quantity2 = 3;
let total = price2 * quantity2; // 8.97
```

<Callout type="tip">You may see `0.1 + 0.2` give `0.30000000000000004`. That’s floating‑point quirks—format for display later, don’t stress now.</Callout>

*When to use numbers:* counts, prices, scores, measurements.

### Booleans: Your Light Switch Boxes

Booleans answer **yes/no** questions: `true` or `false`.

```js
let isLoggedIn = false;
let totalDue = 41.25;
let isFreeShipping = totalDue >= 35; // true
let isMuted = false;
isMuted = !isMuted; // toggle
```

*When to use:* logged in? dark mode on? checkbox checked?

### Null & Undefined: Empty Boxes

Both mean “no value,” but:

* `null` — empty **on purpose** (you set it)
* `undefined` — not assigned yet (JavaScript’s default)

```js
let selectedItem = null; // will fill later
let nextThing;           // undefined
```

| Empty value | Meaning | Who sets it? |
| :---- | :---- | :---- |
| `null` | “Nothing here yet (intentionally).” | You |
| `undefined` | “No value was assigned.” | JS (default) |

### Technical Vocabulary (Primitives)

| Term | Beginner-friendly definition | Quick example |
| :---- | :---- | :---- |
| **String** | Text inside quotes. | `"Hello"` |
| **Interpolation** | Put values inside a backtick string with `${}`. | `` `Hi ${name}` `` |
| **Number** | A numeric value for math. | `42`, `3.14` |
| **Boolean** | A true/false value. | `true`, `false` |
| **null** | Intentional empty placeholder. | `let item = null` |
| **undefined** | Declared but not assigned yet. | `let x;` |
| **Data type** | The kind of value (string, number, …). | `lives` is a number |

## 4) Working with values (operators)

Arithmetic:

```js
3 + 2; 7 - 4; 2 * 3; 8 / 2; 7 % 3; 2 ** 3;
```

String building:

```js
"Hello, " + name + "!";
`Hello, ${name}!`;
```

Comparisons (produce booleans):

```js
age >= 18;    // true / false
score === 10; // strict equality
count !== 0;  // not equal
```

Logical:

```js
isLoggedIn && isAdmin;
!isGameOver;
```

## 5) Picking the right type

Ask: *What am I doing with this value?* Math → number. Display label → string. Toggle → boolean. Waiting for a future object → `null`.

## 8) Checklist

- Default to `const`; drop to `let` only when you reassign.
- Pick the primitive that fits the usage (math? toggle? label?).
- Prefer template literals for multi‑value strings.
- Use `null` for “waiting for a value”; expect `undefined` before assignment.
- Write boolean names that read clearly (`isOpen`, `hasError`).
