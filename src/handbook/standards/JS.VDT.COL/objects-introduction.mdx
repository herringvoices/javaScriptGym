# Data Types → Objects

We've been storing information in individual boxes (variables). But often those boxes all describe **one thing**. That's where **objects** shine: they bundle related info together so it travels as a single unit.

---

## 1) Start with separate variables (spot the pattern)

Imagine these three values all describe the same person:

```js
let name = "Nick";
let age = 35;
let isSleepy = true;
console.log(name, age, isSleepy);
```

They're clearly about **one thing** (a person), just spread across separate boxes.

## 2) Bundle your variables together

Objects are like a **binder**. They gather information together into pages called **properties**. Each property is a **key–value** pair: the **key** is the colored label on the page; the **value** is the information on that page. Keys help you look things up; values are what you're looking for.

Syntax snapshot (notice the **colons** and **commas**):

```js
const person = {
  name: "Nick",      // key: value
  age: 35,            // commas separate properties
  isSleepy: true      // colon (:) instead of equals (=)
};
```

## 3) Read values with dot notation (addresses for data)

**Dot notation** is the default way to **read** a property: <code>object.key</code>. Think of each dot as a step in an **address**—start at the binder, then open the tab you want.

```js
console.log(person.name);     // "Nick"
console.log(person.age);      // 35
console.log(person.isSleepy); // true
```

## 4) Change, add, delete pages (mutating the binder)

> **But wait — doesn't `const` mean "no changes"?**
> When you define the variable, you create a **box** and put your **binder** inside. With `const`, you're locking the **box→binder link**—you can't replace the binder with a new one later. But you **can** still edit the **pages inside** that binder: update a page, add a page, or remove a page. That's called **mutation**.

### Update (rewrite a page)

```js
person.age = 36;     // rewrite the age page
console.log(person.age); // 36
```

### Add (put in a new page)

```js
person.hobby = "board games";
console.log(person.hobby); // "board games"
```

### Delete (remove a page entirely)

```js
delete person.isSleepy;
console.log(person.isSleepy); // undefined
```

---

## Wrap-up

* Start with separate variables to notice "this is one thing," then bundle them into an **object** (your binder).
* Read with **dot notation** like an address; use **bracket notation** only when names get weird or come from a variable.
* Keep the `const` **box→binder link** stable, and **mutate** the pages inside when data changes in your app.
