# Data Types → String Methods

*Last time we* worked with **arrays** and their end-controls (`push`, `pop`, `unshift`, `shift`). Those are **methods**—functions attached to a value. *This time we’re* giving **strings** their own little toolbox. Arrays aren’t the only data type with useful methods and properties.

Two quick reminders before we start:

* A **property** is information **on** a value (no parentheses). Example: <code>.length</code>.
* A **method** is a function **attached to** a value (uses parentheses). Example: <code>text.trim()</code>. Most string methods **return a new string**—store it if you want to keep the change.

## 1) `.length` — how many characters?

`.length` tells you the number of characters in a string. It’s a **property**, so there are **no parentheses**.

## 2) `.trim()` — remove extra spaces at the ends

User input often comes with leading or trailing spaces. `.trim()` cleans both ends. (There are also one-sided versions, but both-ends is the common win.)

    <pre key="peek" className="m-0 whitespace-pre-wrap">{`// const rawName = "   Nick  ";

## 3) `.toLowerCase()` — normalize case

When you compare or store text, it helps to normalize the casing. `.toLowerCase()` returns a new, all-lowercase version.

<HandbookChallenge
  title="Normalize an email"
  hints={[
    "Make <code>emailInput</code> like <code>\"  NICK@example.com \"</code>.",
    "First <em>trim</em> it, then make it all lowercase.",
    "Store the final result in <code>email</code> and log it.",
    <pre key="peek" className="m-0 whitespace-pre-wrap">{`// const emailInput = "  NICK@example.com ";
// const email = emailInput.trim().toLowerCase();
// console.log(email);`}</pre>,
  ]}
  answers={ <pre className="m-0 whitespace-pre-wrap">{`const emailInput = "  NICK@example.com ";

}
>

  <p className="m-0">
    Chaining lets you apply multiple steps in order: first trim, then lower.
  </p>
</HandbookChallenge>

`.split()` takes a delimiter (like a space or comma) and returns an **array** of pieces.

## 5) `.replace(searchValue, replaceValue)` — swap the first match

`.replace()` finds the **first** occurrence of a substring and swaps it.

---

> **What does a method “give back”?**
> Just like <code>const removed = songs.pop()</code> returns the item it removed, most string methods return a **new string**. They don’t change the original; they hand you a transformed copy. Store it if you need it.

You’ll receive three raw strings. Your job: clean and normalize them using only the tools above.

## Wrap-up

* **Property:** <code>.length</code> (no parentheses) tells you how many characters you have.
* **Methods:** <code>.trim()</code>, <code>.toLowerCase()</code>, <code>.split()</code>, <code>.replace()</code> return **new strings** (or, for <code>split</code>, a new array).
* Chain steps in a sensible order (e.g., trim before lowercasing) and store results you want to keep.
